procedure, public vibrations::frf_fitting_tool::fit ( )

Utilizes a relaxed vector fitting algorithm to fit a state-space model the supplied FRF.

Syntax
subroutine fit(class(frf_fitting_tool) this, real(real64) freq(:), real(real64) amp(:), real(real64) phase(:), integer(int32) order, optional integer(int32) niter, optional real(real64) weights(:), optional class(errors) err)
Parameters
[in,out]thisThe frf_fitting_tool object.
[in]freqAn N-element array containing the frequency values at which the frequency response function is defined, in radians/sec.
[in]ampAn N-element array containing the amplitude of the frequency response. Notice, this input must not be log scaled (e.g. in units of dB).
[in]phaseAn N-element array containing the phase of the frequency response, in radians.
[in]orderThe order of the system (must be at least 2).
[in]niterAn optional input defining the number of iterations to perform. The default value is 5.
[in]weightsAn optional N-element array that can be used to weight values at specific frequencies. The default is unity such that no bias is shown.
[in,out]errAn optional errors-based object that if provided can be used to retrieve information relating to any errors encountered during execution. If not provided, a default implementation of the errors class is used internally to provide error handling. Possible errors and warning messages that may be encountered are as follows.
  • MECH_OUT_OF_MEMORY_ERROR: Occurs if insufficient memory is available.
  • MECH_ARRAY_SIZE_ERROR: Occurs if amp, phase, or weights are not sized appropriately.
  • MECH_INVALID_INPUT_ERROR: Occurs if order or niter are invalid.
